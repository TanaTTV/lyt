All 7 fixes — ordered by impact

  • 1
    Lazy-load interactive.js (readline/promises) 10–30 ms startup
    src/cli.js — import() only inside if (wantsInteractive). The most expensive cold-start import, skipped on every direct URL invocation.
  • 2
    Memoize ensureCommand ~20–50 ms saved
    src/cli.js — Map cache in front of spawnSync("--version"). Interactive mode used to fork the process twice; now it forks once and returns the cached executable path.
  • 3
    Chunk array → join() in listFormats O(n²) → O(n)
    src/formats.js — collect stdout chunks in an array and call join("") once on close. Eliminates O(n²) string copying on YouTube's 200–500 KB info JSON.
  • 4
    Batch render writes — N+1 → 1 per frame syscall reduction
    src/progress.js — build the entire render frame (cursor-up + all bars) as one string, call out.write() once. Was N+1 write calls per progress event.
  • 5
    Lazy-load formats.js
    src/cli.js — imported on demand in two places: inside if (options.listFormats) and parallel with interactive.js. Common download path never loads it.
  • 6
    Pre-compute label name in progress state
    src/progress.js — label.padEnd(28) and slice/truncation computed once at state creation, stored as name. formatBar reads the pre-built string with zero allocation.
  • 7
    startsWith before trim() in feed()
    src/cli.js — check line.startsWith("[download]") (no allocation) before line.trim() (allocates). Progress lines — the majority of output in renderer mode — now exit without ever calling trim.
